# webdeck — 움직이는 웹 덱 만들기

파워포인트 대신 **HTML 한 파일**로 끝나는 덱. 브리프만 주면 브랜드 톤에 맞게 조판해 자체완결 덱을 만든다.
> 참고 감도: 구본혁 '후킹' 제안서 웹덱 스타일(대형 타이포·풀블리드·컨셉 애니·조직도). 이미지 퀄리티는 별도 이미지 생성 도구가 없어도, **레이아웃·타이포·애니로 최대한 비슷하게** 낸다.

## 프로세스
1. **내용 받기** — 제목·섹션·핵심 메시지·브랜드 톤(밝음/어둡고 고급/위트)·이미지 유무. 없으면 톤부터 확정.
2. **톤 → theme 결정** — 밝은 브랜드면 `light:true`+밝은 `deckbg`, 어둡고 고급이면 딥톤. accent는 브랜드 컬러.
3. **brief.json 작성** — `brief_template.json`을 복제해 슬라이드를 채운다. 슬라이드 타입은 `references/slide-types.md` 참조. **한 장에 메시지 하나**, 헤드라인은 짧고 굵게.
4. **생성** — `python3 generate_deck.py brief.json 출력폴더` → `index.html`+`deck.config.js`+`assets/`(이미지 자동 복사·상대경로화). 출력폴더의 `index.html`을 브라우저로 열면 끝.
5. **자체완결 공유본(선택)** — 이미지·영상을 data URI로 인라인해 HTML 한 파일로 만들면 어디서든 링크·오프라인으로 열린다.

## 핵심 규칙 (지키면 '있어 보인다')
- **톤앤매너가 전부 결정** — 밝은 브랜드를 다크로 몰지 말 것. 카피 서체·컬러·이미지 무드를 톤에 맞춘다.
- **한국어 줄바꿈은 단어 단위**(렌더러에 `word-break:keep-all` 적용됨) — 어색하게 잘리지 않게.
- **박스(카드)는 균형 배열** — 5개면 3+2, 마지막 하나 덩그러니 금지(렌더러가 자동, `cols`로 강제 가능).
- **이미지·영상은 화면 꽉 채우기**(cover/kv/feature 타입), 중앙 조막만한 이미지 금지.
- **컨셉은 애니로** — 반복/대비/전후 같은 메시지는 `loophell`·`vonr`·`ba` 타입으로 '움직여서' 보여준다.
- 우측 상단 컨트롤(소리 토글·PDF 저장)은 렌더러에 기본 내장.

## 슬라이드 타입 요약 (상세: references/slide-types.md)
`cover`(풀블리드 표지) · `core`(대형 헤드라인, `mood:"light"`로 환기) · `cards`(카드 나열) · `stat-trio`(숫자 3개) · `feature`(좌 텍스트+우 큰 이미지) · `grid`(이미지 그리드, `big`) · `ba`(전후 드래그) · `flow`(단계 플로우) · `orgchart`(조직도) · `loophell`(반복 애니) · `vonr`(낯선 한 끗 애니) · `video`(영상, `sound:true`) · `showcase`(브라우저 목업) · `ending`.

## 이 스킬이 안 하는 것
- AI 이미지·영상 '생성'은 안 한다(별도 도구). 여기선 **주어진 이미지 + 레이아웃·타이포·애니**로 완성한다.
- 이미지가 없으면, 이미지 없이도 강한 **타이포 중심 덱**으로 낸다.
